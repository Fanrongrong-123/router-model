## 路由
**分发请求的（东西）对象就是路由**

## hash模式
- hash: url后面#标识符引起的字符
- 当url后面的hash标识符更改时，将触发hashchange事件
- 路由表：将页面获取字段做一个Hash表，遍历获取
- 适合任何情况

缺点：SEO不友好（浏览器会自动消掉#后面的内容，服务器收不到hash的内容）

## history模式

 在后端将前端所有页面渲染到同一页面包括（404页面）时用

 ## memory模式
 - 将路径存在localstorage里面
 - url不会变，相应变的是localstorage里存的路径
 - 适合非浏览器使用

缺点：没有具体的url，单机有效不能分享
